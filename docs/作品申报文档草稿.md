# 作品申报文档草稿

## 作品名称

别让电池失控：锂电池热失控安全科普互动页

## 作品类别

数字科普作品 - H5 科普互动作品

## 作品简介（300字以内）

《别让电池失控》是一款面向大学生与社会公众的锂电池热失控安全科普 H5 互动作品。作品围绕手机、充电宝、电动车等常见锂电池使用场景，通过风险场景选择、温度滑杆模拟、应急处理问答和学习总结生成等交互环节，帮助用户理解“普通发热”与“热失控风险”之间的区别，掌握鼓包、异味、异常发烫等危险信号的识别方法。作品以浏览器静态页面形式实现，可兼容电脑端与移动端，便于在线传播和离线展示。其核心目标是把相对专业的电化学与安全知识转化为可视化、可操作、可记忆的公众科普内容，提升公众在日常生活中安全使用锂电池产品的意识。

## 科学原理及内容

锂离子电池在正常工作时，通过锂离子在正负极之间迁移完成充放电。异常使用条件下，如过充、短路、挤压、穿刺、高温环境或电池老化，可能导致内部副反应加剧。温度升高会进一步促进材料分解、电解液反应、隔膜损伤等过程，使电池从可控发热转向不可控的连锁升温。该过程即通常所说的热失控。作品将这一过程拆解为“轻微发热—风险积累—危险信号—应急处理”四个层级，帮助用户建立风险判断框架。

## 创作目的

本作品希望解决公众对锂电池安全的两个常见误区：一是把所有发热都视为事故，二是把鼓包、异味、异常发烫等危险信号当作普通老化现象。作品通过互动页面降低理解门槛，使用户在较短时间内掌握基本判断方法和安全处置原则。

## 设计思路

作品采用“问题导入—场景判断—原理解释—应急选择—总结生成”的线性叙事结构。首页通过视觉化电池图形建立主题识别；风险判断模块用日常场景降低距离感；温度滑杆模块用动态反馈呈现风险递进；应急选择模块强化安全行为；总结页则把知识转化为可复述的三句话，提高传播效率。

## 作品源代码说明

作品使用 HTML、CSS、JavaScript 编写，为纯静态页面，不依赖后端服务器、数据库或第三方接口。页面入口为 `index.html`，样式文件为 `assets/style.css`，交互逻辑文件为 `assets/app.js`。作品可直接本地打开，也可部署至 GitHub Pages、腾讯云 CloudBase、阿里云 OSS 等静态托管平台。

## 创新点

作品创新点不在于复杂技术，而在于将锂电池热失控这一专业安全问题转化为低门槛互动体验。相比静态图文，H5 页面能够通过选择、滑杆、反馈和总结生成增强参与感；相比长视频，用户可以按自己的节奏浏览，适合校园宣传、社交传播和线下活动展示。
